# 矩阵向量乘法导出头文件设计

## 目标

只对 `main.cpp` 当前 `selectAlgorithm` 选择的算法计算一次：

```cpp
y = A * x
```

其中 `x` 的所有元素固定为：

```cpp
std::complex<double>(1.0, 0.0)
```

然后把本次算法得到的 `x` 和 `y = A * x` 导出到 `txt` 文件。该功能只用于观察当前算法的一次矩阵向量乘法结果，不做 MoM 参考对比，不计算误差，也不进入 GMRES/CGS 迭代。

## 文件一：`Code/MatVecExport.h`

这是建议新增的完整头文件代码。它是 header-only，实现和声明都放在头文件中，便于直接 `#include "MatVecExport.h"` 使用。

```cpp
// MatVecExport.h
#pragma once
#ifndef MATVECEXPORT_H
#define MATVECEXPORT_H

#include <cmath>
#include <complex>
#include <fstream>
#include <iomanip>
#include <iostream>
#include <stdexcept>
#include <string>
#include <vector>

namespace MatVecExport {
    using Complex = std::complex<double>;
    using Vector = std::vector<Complex>;

    struct Result {
        Vector x;
        Vector y;
    };

    inline Vector makeOnesVector(int n)
    {
        if (n <= 0) {
            throw std::invalid_argument(
                "MatVecExport::makeOnesVector: n must be positive.");
        }

        return Vector(static_cast<size_t>(n), Complex(1.0, 0.0));
    }

    template <typename AxFunc>
    inline Vector applyOnce(int n, const Vector& x, AxFunc ax)
    {
        if (n <= 0) {
            throw std::invalid_argument(
                "MatVecExport::applyOnce: n must be positive.");
        }

        if (static_cast<int>(x.size()) != n) {
            throw std::invalid_argument(
                "MatVecExport::applyOnce: x size mismatch.");
        }

        Vector y(static_cast<size_t>(n), Complex(0.0, 0.0));
        ax(n, x.data(), y.data());
        return y;
    }

    inline void exportResultTxt(
        const std::string& filePath,
        const Result& result,
        const std::string& algorithmName,
        int selectAlgorithm,
        int selectIntegralEqu)
    {
        if (result.x.size() != result.y.size()) {
            throw std::invalid_argument(
                "MatVecExport::exportResultTxt: x and y size mismatch.");
        }

        std::ofstream out(filePath);
        if (!out.is_open()) {
            throw std::runtime_error(
                "MatVecExport::exportResultTxt: failed to open output file: "
                + filePath);
        }

        out << std::setprecision(17);
        out << "# MatVec one-shot result\n";
        out << "# algorithm = " << algorithmName << '\n';
        out << "# selectAlgorithm = " << selectAlgorithm << '\n';
        out << "# selectIntegralEqu = " << selectIntegralEqu << '\n';
        out << "# size = " << result.x.size() << '\n';
        out << "# x is fixed to complex<double>(1.0, 0.0)\n";
        out << "# columns: index x_real x_imag y_real y_imag abs_y\n";

        for (size_t i = 0; i < result.x.size(); ++i) {
            out << i << '\t'
                << result.x[i].real() << '\t'
                << result.x[i].imag() << '\t'
                << result.y[i].real() << '\t'
                << result.y[i].imag() << '\t'
                << std::abs(result.y[i]) << '\n';
        }

        out.close();
        std::cout << "Info: one-shot A*x exported to "
            << filePath << " (MatVecExport.h)" << std::endl;
    }

    template <typename Solver>
    inline Result runAndExport(
        Solver& solver,
        int n,
        const std::string& filePath,
        const std::string& algorithmName,
        int selectAlgorithm,
        int selectIntegralEqu)
    {
        Result result;
        result.x = makeOnesVector(n);
        result.y = applyOnce(
            n,
            result.x,
            [&solver](int n,
                const Complex* x,
                Complex* y) {
                    solver.applyAx(n, x, y);
            });

        exportResultTxt(
            filePath,
            result,
            algorithmName,
            selectAlgorithm,
            selectIntegralEqu);

        return result;
    }
}

#endif // MATVECEXPORT_H
```

## 代码讲解

`makeOnesVector(int n)` 只负责生成输入向量 `x`。这里不使用随机数，所有元素都固定为 `(1.0, 0.0)`，所以每次运行同一个模型都会得到同一个输入。

`applyOnce` 是真正执行一次 `A*x` 的函数。它不关心当前算法是 MoM、FMM 还是 MLFMM，只要求传入的 `ax` 满足当前求解器已经使用的签名：

```cpp
void(int n, const std::complex<double>* x, std::complex<double>* y)
```

`exportResultTxt` 负责写文件。输出列为：

```txt
index x_real x_imag y_real y_imag abs_y
```

`runAndExport` 是面向 `main.cpp` 的入口。只要当前算法类提供 `applyAx` 成员函数，就可以直接调用：

```cpp
MatVecExport::runAndExport(solver, n, filePath, algorithmName,
    selectAlgorithm, selectIntegralEqu);
```

## 文件二：给求解器增加 `applyAx`

现有 `MoM::matrix_solver`、`FMM::matrix_solver`、`MLFMM::matrix_solver` 内部已经各自有一个 `ax` lambda。建议把这个 lambda 的函数体提取成公开成员函数 `applyAx`，这样 GMRES/CGS 和单次导出会使用同一份矩阵向量乘法代码。

### `MoM.cpp` 增加完整实现

```cpp
void MoM::applyAx(
    int n,
    const std::complex<double>* x,
    std::complex<double>* w)
{
    if (n <= 0 || x == nullptr || w == nullptr) {
        throw std::invalid_argument("MoM::applyAx invalid input.");
    }

    if (static_cast<int>(Z_mom.size()) != n) {
        throw std::invalid_argument("MoM::applyAx dimension mismatch.");
    }

#pragma omp parallel for
    for (int i = 0; i < n; ++i) {
        if (static_cast<int>(Z_mom[i].size()) != n) {
            throw std::invalid_argument("MoM::applyAx row dimension mismatch.");
        }

        std::complex<double> sum(0.0, 0.0);
        for (int j = 0; j < n; ++j) {
            sum += Z_mom[i][j] * x[j];
        }
        w[i] = sum;
    }
}
```

然后把 `MoM::matrix_solver` 中原来的 `ax` lambda 改成：

```cpp
void MoM::matrix_solver(
    int n,
    std::complex<double> x[],
    std::complex<double> rhs[],
    int itr_max,
    int mr,
    double tol_abs,
    double tol_rel)
{
    auto ax = [this](int n,
        const std::complex<double>* x,
        std::complex<double>* w) {
            this->applyAx(n, x, w);
        };

    bool converged = false;
    switch (matrixSolverType_)
    {
    case 0:
        std::cout << "Matrix solver: GMRES\n";
        converged = MGMRES::mgmres(
            n, x, rhs, itr_max, mr, tol_abs, tol_rel, ax);
        if (!converged) {
            std::cerr << "GMRES did not converge.\n";
        }
        break;

    case 1:
        std::cout << "Matrix solver: CGS\n";
        converged = MCGS::CGS::solve(
            n, x, rhs, itr_max, tol_rel, ax);
        if (!converged) {
            std::cerr << "CGS did not converge.\n";
        }
        break;

    default:
        throw std::invalid_argument(
            "Invalid matrixSolverType_ (must be 0 for GMRES or 1 for CGS)");
    }
}
```

## 文件三：建议增加构造模式

当前 `MoM`、`FMM`、`MLFMM` 构造函数会在构造完成后继续进入 RCS 求解流程。单次导出 `A*x` 时不希望求解 RCS，因此建议增加构造模式。

可以新增一个公共枚举，例如放在新的 `SolverRunMode.h`，也可以先放在三个类共同能访问的头文件里：

```cpp
#pragma once
#ifndef SOLVERRUNMODE_H
#define SOLVERRUNMODE_H

enum class SolverRunMode {
    BuildOnly,
    SolveRCS
};

#endif // SOLVERRUNMODE_H
```

构造函数参数增加默认值：

```cpp
SolverRunMode runMode = SolverRunMode::SolveRCS
```

例如 `MoM` 构造函数声明变成：

```cpp
MoM(
    const RCSExportConfig& cfg,
    const int selectIntegralEqu,
    const int selectMatrixSolver,
    const std::string selectMono_Dual,
    const std::string pol_wave,
    const std::vector<std::vector<OCTree::Node*>>& octreeNodes,
    const std::vector<RWGBase>& rwgs,
    const int maxLevel_,
    const gaussPoints& gausspoint,
    const double E0,
    const EMSource& wave,
    SolverRunMode runMode = SolverRunMode::SolveRCS);
```

构造函数中，在矩阵、近场、远场、工作区初始化完成后加判断：

```cpp
if (runMode == SolverRunMode::BuildOnly) {
    return;
}
```

这样正常运行时不需要改现有调用；只有导出 `A*x` 时显式传入 `SolverRunMode::BuildOnly`。

## `main.cpp` 完整接入示例

先在 `main.cpp` 顶部加入：

```cpp
#include "MatVecExport.h"
#include "SolverRunMode.h"
```

在单算例参数区加入开关：

```cpp
int exportOneShotMatVec = 1; // 0 = off, 1 = on
```

在 `OCTree octree(...)` 构造完成、`selectMatrixSolver` 合法性检查之后、正式进入 `if (selectAlgorithm == 0/1/2)` 求解前，插入：

```cpp
if (exportOneShotMatVec == 1) {
    const int n = (selectIntegralEqu == 2)
        ? 2 * static_cast<int>(rwgBases.size())
        : static_cast<int>(rwgBases.size());

    if (selectAlgorithm == 0) {
        MoM mom(
            cfg, selectIntegralEqu, selectMatrixSolver,
            selectMono_Dual, polarization,
            octree.octreeNodes_, rwgBases, octree.maxLevel_,
            gausspoint, E0, wave,
            SolverRunMode::BuildOnly);

        MatVecExport::runAndExport(
            mom,
            n,
            "matvec_mom.txt",
            "MoM",
            selectAlgorithm,
            selectIntegralEqu);
    }
    else if (selectAlgorithm == 1) {
        FMM fmm(
            cfg, selectIntegralEqu, selectMatrixSolver,
            selectMono_Dual, polarization,
            octree.octreeNodes_, rwgBases, octree.maxLevel_,
            gausspoint, E0, wave,
            SolverRunMode::BuildOnly);

        MatVecExport::runAndExport(
            fmm,
            n,
            "matvec_fmm.txt",
            "FMM",
            selectAlgorithm,
            selectIntegralEqu);
    }
    else if (selectAlgorithm == 2) {
        MLFMM mlfmm(
            cfg, selectIntegralEqu, selectMatrixSolver,
            selectMono_Dual, polarization,
            octree.octreeNodes_, octree.octreeNodesDRvec_,
            rwgBases, octree.maxLevel_, gausspoint,
            E0, wave,
            SolverRunMode::BuildOnly);

        MatVecExport::runAndExport(
            mlfmm,
            n,
            "matvec_mlfma.txt",
            "MLFMA",
            selectAlgorithm,
            selectIntegralEqu);
    }
    else {
        std::cerr << "Error: Invalid selectAlgorithm value: "
            << selectAlgorithm << std::endl;
        return 3;
    }

    return 0;
}
```

## 输出文件格式

以 `selectAlgorithm == 2` 为例，输出文件 `matvec_mlfma.txt` 内容类似：

```txt
# MatVec one-shot result
# algorithm = MLFMA
# selectAlgorithm = 2
# selectIntegralEqu = 1
# size = 128
# x is fixed to complex<double>(1.0, 0.0)
# columns: index x_real x_imag y_real y_imag abs_y
0    1    0    ...
1    1    0    ...
```

其中：

1. `index` 是向量下标。
2. `x_real`、`x_imag` 是输入向量元素。
3. `y_real`、`y_imag` 是当前算法一次 `A*x` 的输出。
4. `abs_y` 是 `std::abs(y[i])`，方便快速看幅度。

## 维度规则

PEC EFIE/CFIE：

```cpp
n = row
```

PMCHWT：

```cpp
n = 2 * row
```

PMCHWT 继续沿用现有未知量顺序：

```cpp
x[0:N)   = electric current unknowns
x[N:2N) = magnetic current unknowns
```

## 最小改动清单

1. 新增 `Code/MatVecExport.h`。
2. 新增 `Code/SolverRunMode.h`。
3. 给 `MoM`、`FMM`、`MLFMM` 增加公开 `applyAx`。
4. 把三个 `matrix_solver` 中原本的 `ax` lambda 函数体移动到对应 `applyAx`。
5. 让三个 `matrix_solver` 通过 `this->applyAx(...)` 调用矩阵向量乘法。
6. 给三个构造函数增加 `SolverRunMode runMode = SolverRunMode::SolveRCS`。
7. 在 `main.cpp` 增加 `exportOneShotMatVec` 分支，只构造当前算法并导出一次 `A*x`。
