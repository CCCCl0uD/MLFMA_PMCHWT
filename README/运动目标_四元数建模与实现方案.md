# 运动目标的四元数建模与实现方案

本文档给出在当前 MoM、FMM、MLFMM 框架中加入运动目标计算的实现方案。目标是在不破坏现有 PEC/介质目标求解流程的前提下，引入刚体平移、刚体旋转、时间步序列和可选的多普勒相位修正。

当前程序的主流程是：

```text
NAS 网格 -> RWG 基函数 -> OCTree -> MoM/FMM/MLFMM 求解 -> RCS 输出
```

运动目标建议新增在 **NAS 网格读取之后、RWG 基函数生成之前**：

```text
NAS 原始网格
  -> 根据 t 计算目标姿态 q(t) 与位置 c(t)
  -> 变换 points/faces
  -> RWG 基函数
  -> OCTree
  -> MoM/FMM/MLFMM
  -> 当前时刻 RCS/回波输出
```

这样 MoM、FMM、MLFMM 的矩阵装配和求解代码几乎不用改，运动目标只作为“每个时刻的一套几何状态”进入已有求解器。

---

## 1. 建模选择

### 1.1 推荐路线：几何变换后重新求解

对第 `n` 个时间步：

```math
\mathbf r_n = \mathbf c(t_n) + \mathbf R(q(t_n))(\mathbf r_0 - \mathbf c_0)
```

其中：

- `r0` 是原始网格点。
- `c0` 是旋转中心，通常取目标质心或用户指定参考点。
- `c(t)` 是目标在世界坐标系中的平移位置。
- `q(t)` 是目标姿态四元数。
- `R(q)` 是四元数对应的旋转矩阵。

优点：

- 对 PEC、介质、MoM、FMM、MLFMM 都一致。
- 对任意姿态、任意运动轨迹都可靠。
- 法向量、面积、八叉树、近远场关系都自然随目标状态更新。

缺点：

- 每个时间步都要重新生成 RWG、OCTree、近场矩阵和预处理矩阵。
- 计算量较大，但实现最稳。

这是第一阶段建议采用的方案。

### 1.2 快速路线：固定目标，旋转入射/散射方向

如果目标只做绕固定中心的刚体旋转，且不考虑平移相位/多普勒，可以把目标保持不动，把入射方向、散射方向旋转到目标体坐标系：

```math
\hat{\mathbf k}' = R(q)^{-1}\hat{\mathbf k}
```

优点：

- 不需要重建 RWG、OCTree、矩阵。
- 适合快速生成姿态角 RCS 数据库。

缺点：

- 不适合平移目标的绝对相位。
- 对多目标、非原点旋转、宽带回波不够直接。
- 当前 `RCS.h` 的扫角逻辑以角度输入为主，需要增加方向向量版接口。

该路线适合作为第二阶段优化。

## 2. 四元数基础

四元数写作：

```math
q = w + xi + yj + zk
```

单位四元数表示旋转。对向量 `v` 的旋转为：

```math
\mathbf v' = q \mathbf v q^{-1}
```

若角速度向量为：

```math
\boldsymbol\omega = [\omega_x,\omega_y,\omega_z]
```

则姿态微分方程为：

```math
\dot q = \frac{1}{2} q \otimes [0,\boldsymbol\omega]
```

工程实现中可以用指数映射进行一步更新：

```math
q(t+\Delta t) = q(t)\otimes q_\Delta
```

```math
q_\Delta =
\left[
\cos\frac{\theta}{2},
\hat{\boldsymbol\omega}\sin\frac{\theta}{2}
\right],
\quad
\theta = |\boldsymbol\omega|\Delta t
```

## 3. 新增文件：Motion.h

建议新增 `Code/Motion.h`，专门放运动目标的数据结构、四元数和网格刚体变换函数。

```cpp
// Motion.h
#pragma once
#ifndef MOTION_H
#define MOTION_H

#include "MyStruct.h"

#include <cmath>
#include <vector>

namespace Motion {

struct Vec3 {
	double x = 0.0;
	double y = 0.0;
	double z = 0.0;
};

struct Quaternion {
	double w = 1.0;
	double x = 0.0;
	double y = 0.0;
	double z = 0.0;

	static Quaternion identity() {
		return {};
	}

	static Quaternion fromAxisAngle(const Vec3& axis, double angleRad) {
		const double n = std::sqrt(axis.x * axis.x + axis.y * axis.y + axis.z * axis.z);
		if (n <= 1.0e-15) {
			return identity();
		}

		const double half = 0.5 * angleRad;
		const double s = std::sin(half) / n;
		Quaternion q;
		q.w = std::cos(half);
		q.x = axis.x * s;
		q.y = axis.y * s;
		q.z = axis.z * s;
		q.normalize();
		return q;
	}

	static Quaternion fromAngularVelocity(const Vec3& omegaRadPerSec, double dt) {
		const double omegaNorm = std::sqrt(
			omegaRadPerSec.x * omegaRadPerSec.x +
			omegaRadPerSec.y * omegaRadPerSec.y +
			omegaRadPerSec.z * omegaRadPerSec.z);

		if (omegaNorm <= 1.0e-15 || dt == 0.0) {
			return identity();
		}

		Vec3 axis{
			omegaRadPerSec.x / omegaNorm,
			omegaRadPerSec.y / omegaNorm,
			omegaRadPerSec.z / omegaNorm
		};
		return fromAxisAngle(axis, omegaNorm * dt);
	}

	void normalize() {
		const double n = std::sqrt(w * w + x * x + y * y + z * z);
		if (n <= 1.0e-15) {
			w = 1.0;
			x = y = z = 0.0;
			return;
		}
		w /= n;
		x /= n;
		y /= n;
		z /= n;
	}

	Quaternion conjugate() const {
		return { w, -x, -y, -z };
	}

	Quaternion operator*(const Quaternion& b) const {
		return {
			w * b.w - x * b.x - y * b.y - z * b.z,
			w * b.x + x * b.w + y * b.z - z * b.y,
			w * b.y - x * b.z + y * b.w + z * b.x,
			w * b.z + x * b.y - y * b.x + z * b.w
		};
	}

	Vec3 rotate(const Vec3& v) const {
		const Quaternion p{ 0.0, v.x, v.y, v.z };
		const Quaternion r = (*this) * p * conjugate();
		return { r.x, r.y, r.z };
	}
};

struct RigidMotionState {
	double time = 0.0;
	Vec3 translation{};
	Vec3 velocity{};
	Vec3 rotationCenter{};
	Quaternion attitude{};
	Vec3 angularVelocity{};
};

inline Vec3 toVec3(const Point& p) {
	return { p.x, p.y, p.z };
}

inline Point toPoint(int id, const Vec3& v) {
	return { id, v.x, v.y, v.z };
}

inline Vec3 add(const Vec3& a, const Vec3& b) {
	return { a.x + b.x, a.y + b.y, a.z + b.z };
}

inline Vec3 sub(const Vec3& a, const Vec3& b) {
	return { a.x - b.x, a.y - b.y, a.z - b.z };
}

inline Vec3 scale(const Vec3& a, double s) {
	return { a.x * s, a.y * s, a.z * s };
}

inline Vec3 cross(const Vec3& a, const Vec3& b) {
	return {
		a.y * b.z - a.z * b.y,
		a.z * b.x - a.x * b.z,
		a.x * b.y - a.y * b.x
	};
}

inline double norm(const Vec3& a) {
	return std::sqrt(a.x * a.x + a.y * a.y + a.z * a.z);
}

inline Vec3 normalize(const Vec3& a) {
	const double n = norm(a);
	if (n <= 1.0e-15) {
		return {};
	}
	return { a.x / n, a.y / n, a.z / n };
}

inline double triangleArea(const Vec3& a, const Vec3& b, const Vec3& c) {
	const Vec3 ab = sub(b, a);
	const Vec3 ac = sub(c, a);
	return 0.5 * norm(cross(ab, ac));
}

inline Vec3 triangleCenter(const Vec3& a, const Vec3& b, const Vec3& c) {
	return {
		(a.x + b.x + c.x) / 3.0,
		(a.y + b.y + c.y) / 3.0,
		(a.z + b.z + c.z) / 3.0
	};
}

inline Vec3 triangleNormal(const Vec3& a, const Vec3& b, const Vec3& c) {
	const Vec3 ab = sub(b, a);
	const Vec3 ac = sub(c, a);
	return normalize(cross(ab, ac));
}

inline Point transformPoint(const Point& p, const RigidMotionState& state) {
	const Vec3 r0 = toVec3(p);
	const Vec3 local = sub(r0, state.rotationCenter);
	const Vec3 rotated = state.attitude.rotate(local);
	const Vec3 moved = add(add(state.rotationCenter, rotated), state.translation);
	return toPoint(p.id, moved);
}

inline std::vector<Point> transformPoints(
	const std::vector<Point>& sourcePoints,
	const RigidMotionState& state)
{
	std::vector<Point> result;
	result.reserve(sourcePoints.size());
	for (const Point& p : sourcePoints) {
		result.push_back(transformPoint(p, state));
	}
	return result;
}

inline Point* findPointById(std::vector<Point>& points, int id) {
	for (Point& p : points) {
		if (p.id == id) {
			return &p;
		}
	}
	return nullptr;
}

inline std::vector<FaceElement> rebuildFacesOnTransformedPoints(
	const std::vector<FaceElement>& sourceFaces,
	std::vector<Point>& transformedPoints)
{
	std::vector<FaceElement> result;
	result.reserve(sourceFaces.size());

	for (const FaceElement& f : sourceFaces) {
		FaceElement nf;
		nf.faceId = f.faceId;
		nf.vertex1 = findPointById(transformedPoints, f.vertex1->id);
		nf.vertex2 = findPointById(transformedPoints, f.vertex2->id);
		nf.vertex3 = findPointById(transformedPoints, f.vertex3->id);

		const Vec3 a = toVec3(*nf.vertex1);
		const Vec3 b = toVec3(*nf.vertex2);
		const Vec3 c = toVec3(*nf.vertex3);
		const Vec3 n = triangleNormal(a, b, c);
		const Vec3 center = triangleCenter(a, b, c);

		nf.externalNormalVector = toPoint(0, n);
		nf.triangularArea = triangleArea(a, b, c);
		nf.triangleCenter = toPoint(0, center);
		result.push_back(nf);
	}

	return result;
}

inline RigidMotionState advanceConstantVelocity(
	const RigidMotionState& initial,
	double time)
{
	RigidMotionState state = initial;
	state.time = time;
	state.translation = add(initial.translation, scale(initial.velocity, time));
	const Quaternion dq = Quaternion::fromAngularVelocity(initial.angularVelocity, time);
	state.attitude = dq * initial.attitude;
	state.attitude.normalize();
	return state;
}

} // namespace Motion

#endif // MOTION_H
```

## 4. 在单次 main.cpp 中接入

当前 `main.cpp` 是一次读取、一次求解。如果要做运动目标时间序列，建议保留原始 `points/faces`，每个时刻生成一份变换后的副本。

### 4.1 引入头文件

```cpp
#include "Motion.h"
```

### 4.2 读取原始网格后保存基准几何

在 `readNasData()` 之后：

```cpp
std::vector<Point> basePoints = points;
std::vector<FaceElement> baseFaces = faces;
```

注意：`baseFaces` 中的 `vertex1/2/3` 指针指向 `points`，因此后续不能直接拿它和变换后的 `points` 混用。要通过 `rebuildFacesOnTransformedPoints()` 重新绑定指针。

### 4.3 定义运动参数

例如目标以 `200 m/s` 沿 `x` 方向平移，同时绕 `z` 轴以 `30 deg/s` 旋转：

```cpp
Motion::RigidMotionState initialMotion;
initialMotion.rotationCenter = { 0.0, 0.0, 0.0 };
initialMotion.translation = { 0.0, 0.0, 0.0 };
initialMotion.velocity = { 200.0, 0.0, 0.0 };
initialMotion.attitude = Motion::Quaternion::identity();
initialMotion.angularVelocity = { 0.0, 0.0, 30.0 * Pi / 180.0 };

const double tStart = 0.0;
const double tEnd = 0.01;
const double dt = 0.001;
```

### 4.4 每个时间步重新生成几何并求解

把原来的一次性：

```cpp
RWGGenerator::GenerateRWG(faces, points, rwgBases);
OCTree octree(rwgBases, wave.wavelength(), inputFile);
MoM/FMM/MLFMM solver(...);
```

改成时间循环：

```cpp
for (int ti = 0; ; ++ti) {
	const double t = tStart + ti * dt;
	if (t > tEnd + 1.0e-12) {
		break;
	}

	Motion::RigidMotionState state =
		Motion::advanceConstantVelocity(initialMotion, t);

	std::vector<Point> movingPoints =
		Motion::transformPoints(basePoints, state);

	std::vector<FaceElement> movingFaces =
		Motion::rebuildFacesOnTransformedPoints(baseFaces, movingPoints);

	std::vector<RWGBase> movingRWG;
	if (!RWGGenerator::GenerateRWG(movingFaces, movingPoints, movingRWG)) {
		std::cerr << "Error: moving RWG generation failed at t = "
			<< t << std::endl;
		continue;
	}

	OCTree movingTree(movingRWG, wave.wavelength(), inputFile);

	RCSExportConfig movingCfg = cfg;
	movingCfg.setFixedTimeStamp(
		"t" + std::to_string(static_cast<int>(std::round(t * 1.0e6))) + "us");

	if (selectAlgorithm == 0) {
		MoM mom(movingCfg, selectIntegralEqu, selectMatrixSolver,
			selectMono_Dual, polarization,
			movingTree.octreeNodes_, movingRWG, movingTree.maxLevel_,
			gausspoint, E0, wave);
	}
	else if (selectAlgorithm == 1) {
		FMM fmm(movingCfg, selectIntegralEqu, selectMatrixSolver,
			selectMono_Dual, polarization,
			movingTree.octreeNodes_, movingRWG, movingTree.maxLevel_,
			gausspoint, E0, wave);
	}
	else if (selectAlgorithm == 2) {
		MLFMM mlfmm(movingCfg, selectIntegralEqu, selectMatrixSolver,
			selectMono_Dual, polarization,
			movingTree.octreeNodes_, movingTree.octreeNodesDRvec_,
			movingRWG, movingTree.maxLevel_, gausspoint, E0, wave);
	}
}
```

这个版本会为每个时刻输出一个独立 RCS 文件。后续可以把输出聚合到一个 `_MovingRCS.txt` 文件中。

## 5. BatchRunner 接入方案

建议在 `SimTask` 中新增运动参数：

```cpp
bool moving = false;
double tStart = 0.0;
double tEnd = 0.0;
double dt = 0.0;

double vx = 0.0, vy = 0.0, vz = 0.0;
double wxDeg = 0.0, wyDeg = 0.0, wzDeg = 0.0;
double cx = 0.0, cy = 0.0, cz = 0.0;
```

配置文件中新增：

```text
moving = 1
t_start = 0.0
t_end = 0.01
dt = 0.001
velocity = 200.0, 0.0, 0.0
angular_velocity_deg = 0.0, 0.0, 30.0
rotation_center = 0.0, 0.0, 0.0
```

在 `BatchRunner::runSingleTask()` 中：

- 如果 `moving == false`，保持现有流程。
- 如果 `moving == true`，先读原始网格，然后对每个时间步执行“变换网格 -> RWG -> OCTree -> 求解”。

伪代码：

```cpp
if (!task.moving) {
	// existing flow
}
else {
	std::vector<Point> basePoints;
	std::vector<FaceElement> baseFaces;
	readNasData(task.nasFile, basePoints, baseFaces);

	Motion::RigidMotionState initial;
	initial.rotationCenter = { task.cx, task.cy, task.cz };
	initial.velocity = { task.vx, task.vy, task.vz };
	initial.angularVelocity = {
		task.wxDeg * Pi / 180.0,
		task.wyDeg * Pi / 180.0,
		task.wzDeg * Pi / 180.0
	};

	for (double t = task.tStart; t <= task.tEnd + 1.0e-12; t += task.dt) {
		auto state = Motion::advanceConstantVelocity(initial, t);
		auto movingPoints = Motion::transformPoints(basePoints, state);
		auto movingFaces = Motion::rebuildFacesOnTransformedPoints(baseFaces, movingPoints);
		// Generate RWG, build tree, solve h/v polarization.
	}
}
```

## 6. 多普勒与回波相位

如果只输出每个时刻的 RCS，刚体位置变换已经足够。若要生成运动目标回波，需要额外考虑相位和多普勒。

### 6.1 平移相位

远场平面波下，目标整体平移 `c(t)` 会带来入射与散射方向相关的相位：

```math
\phi(t) = -k \hat{\mathbf k}_{inc}\cdot \mathbf c(t)
          +k \hat{\mathbf k}_{sca}\cdot \mathbf c(t)
```

双站散射场可写成：

```math
E_s(t) = E_{body}(t)\exp(j\phi(t))
```

单站时，若程序中 `kinc` 指向波传播方向，单站相位近似为：

```math
\phi(t) \approx -2k \hat{\mathbf k}_{inc}\cdot \mathbf c(t)
```

### 6.2 多普勒频移

双站多普勒：

```math
f_D =
\frac{1}{\lambda}
(\hat{\mathbf k}_{sca} - \hat{\mathbf k}_{inc})\cdot \mathbf v
```

单站近似：

```math
f_D \approx -\frac{2}{\lambda}\hat{\mathbf k}_{inc}\cdot \mathbf v
```

旋转目标还会产生微多普勒。对于目标面元位置：

```math
\mathbf r(t)=\mathbf c(t)+R(q(t))(\mathbf r_0-\mathbf c_0)
```

速度为：

```math
\mathbf v(\mathbf r,t)
=
\mathbf v_c(t)
+
\boldsymbol\omega(t)\times
\left[
\mathbf r(t)-\mathbf c(t)
\right]
```

微多普勒来自各散射中心的速度投影差异。第一阶段建议先输出时间序列 RCS；第二阶段再把 `RCS_pre` 作为复数回波序列，做 FFT 得到多普勒谱。

## 8. 推荐实施顺序

### 第一阶段：稳态时间序列

1. 新增 `Motion.h`。
2. 在 `main.cpp` 中实现单目标时间步循环。
3. 每个时间步重建 `points/faces/rwg/octree/solver`。
4. 输出每个时刻 RCS 文件。

这一阶段可以验证运动姿态是否正确。

### 第二阶段：批处理

1. 扩展 `BatchRunner.h` 的配置解析。
2. 支持 `moving = 1`。
3. 自动对 `h/v` 极化和多个时间步循环。
4. 输出统一的时间序列表格。

### 第三阶段：回波与多普勒

1. 在 `RCS.h` 中增加复数散射场输出接口。
2. 加入平移相位修正。
3. 输出 `E_v(t), E_h(t)`。
4. 使用外部脚本或 C++ FFT 计算多普勒谱。

## 9. 重要注意事项

### 9.1 不要原地修改原始 points

`FaceElement` 和 `RWGBase` 内部有大量指针。如果直接原地改 `points`，虽然短期能工作，但很容易在多时间步中混淆基准几何和当前几何。建议始终保留：

```cpp
basePoints
baseFaces
```

每个时刻生成：

```cpp
movingPoints
movingFaces
movingRWG
movingTree
```

### 9.2 每个时间步必须重建 RWG

RWG 基函数里保存了：

- 正负面指针
- 自由顶点指针
- 公共边顶点指针
- 边长
- 边中心

如果网格坐标变了，而 RWG 不重建，这些派生量会过期。

### 9.3 每个时间步必须重建 OCTree

FMM/MLFMM 的近远场关系依赖目标空间位置。平移后如果不重建八叉树，近场矩阵和远场聚合中心都会不一致。

### 9.4 旋转中心要固定清楚

若目标绕自身质心旋转，先计算质心：

```math
\mathbf c_0 = \frac{1}{N}\sum_i \mathbf r_i
```

若目标绕雷达坐标系某个点旋转，则直接指定该点。

### 9.5 角速度单位必须用 rad/s

用户配置可以用 `deg/s`，但进入 `Motion::RigidMotionState` 前必须转换：

```cpp
omegaRad = omegaDeg * Pi / 180.0;
```

## 11. 推荐结论

对当前工程，最稳的实现方式是：

```text
四元数负责姿态
速度负责平移
每个时间步变换 NAS 网格副本
重新生成 RWG 和 OCTree
复用现有 MoM/FMM/MLFMM 求解器
输出 RCS 时间序列
```

这条路线代码侵入最小，也最容易验证。等时间序列 RCS 正确后，再继续做快速姿态库、多普勒相位和微多普勒谱。
